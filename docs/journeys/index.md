# Getting Started Journeys

Welcome to the AMD Instinct Getting Started Guide! Choose the deployment path that best fits your needs and environment.

## Choose Your Journey

We've designed two comprehensive journeys to help you get started with AMD Instinct GPUs, each tailored to different deployment scenarios and requirements.

### 🖥️ [Bare Metal Journey](bare-metal/index.md)

Deploy AMD Instinct GPUs directly on your hardware infrastructure for maximum performance and control.

**Best for:**
- Enterprise AI infrastructure
- High-performance computing clusters
- Production inference environments
- Custom hardware configurations
- Maximum performance requirements

**What you'll learn:**
- Hardware selection and compatibility
- ROCm installation and system configuration
- Environment optimization
- Performance validation and tuning

### ☁️ [Cloud Journey](cloud/index.md)

Deploy AMD Instinct GPUs in cloud environments for flexibility, scalability, and reduced infrastructure management.

**Best for:**
- Rapid prototyping and experimentation
- Variable workload demands
- Distributed teams and collaboration
- Hybrid cloud/on-premises environments
- Minimal infrastructure management

**What you'll learn:**
- Cloud provider selection and comparison
- Instance provisioning and configuration
- Cloud-optimized environment setup
- Integration with cloud services

## Journey Structure

Each journey follows a structured, step-by-step approach:

1. **Planning & Selection** - Choose the right hardware or cloud provider
2. **Installation & Provisioning** - Set up your environment
3. **Environment Configuration** - Optimize your setup
4. **Validation & Testing** - Verify everything works correctly
5. **Optimization** - Fine-tune for peak performance

## Getting Started

Ready to begin? Click on your preferred journey above to start your AMD Instinct deployment experience. Each journey includes practical examples, troubleshooting tips, and best practices learned from real-world deployments.
